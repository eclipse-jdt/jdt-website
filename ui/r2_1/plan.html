Overall 2.1 Plan
The 2.1 plan below is a proto plan and input from the community is very welcome.

The overall themes for JUI are:

  • reduce gestures, improve navigation
  • more refactoring support
  • smarter editing

2.1 Development Effort - Milestone 4 (December 13th, 2002)

JDT UI

  • General Issues:
    • address NotPresentExcpetion and validateEdit
    • address keybinding support in JDT-UI (bug 26047) 
    • assign short cuts to frequently used actions (e.g rename, extract local, ...)
    • address accessibility issue of preference pages use groups instead of label headers (bug 26186)
    • enable GoTo / Open Resource in Java perspective (bug 26223)
    • resolve back linking of editors with views
  • Performance pass:
    • test typically scenarios regarding its performance
      • starting-up a large workspace
      • opening the first Java perspective
      • opening the first Java Editor
      • selection change in PackageExplorer (single and multiple selection)
    • memory consumption
  • Wizards:
    • add "New Class" drop down to allow to create classes, interfaces, TestCases, visual classes, ...
  • Package Explorer:
    • render Java files outside of the build path differently (important for exclusion support)
    • provide filter to hide java compilation units that are excluded
    • introduce a library folder (container for referenced JARs and class path containers)
  • Java Browsing:
    • support logical packages (e.g. combine the contents multiple package fragments with the same name into a single logical package). Only a limited set of actions will be available on a logical package.
  • Build path:
    • support to exclude folders
    • container improvements
    • linked output folders
      • depends on standard resource creation dialog from workbench
  • Refactoring:
    • polymorphic inline action
    • enable Edit> undo for single file refactorings
    • enable contributions to Refactoring menu
    • provide API to execute refactorings headless (at least for rename and move)
    • investigate lightweight wizard-dialog
    • Enhance existing refactorings
      • Push Up member
        • allow pushing up members to indirect superclass.
        • help with pushing up dependent members
      • Extract interface
        • enhance to include constants
        • enhance to work on interfaces
      • Change method signature
        • enable for no-arg constructors
  • Code generation:
    • improved name guessing of variables
  • Quick assist
    • assist overriding/implementing a method
  • Compare
    • UI improvements
      • use splines to connect difference ranges
      • indicate status for number of incoming/outgoing changes
      • indicate resolution of a change
    • auto merge: accept all incoming
    • surface additional API
