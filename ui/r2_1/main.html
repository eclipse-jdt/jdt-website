  • for R2.1, the major focus points were:
    • Performance
      • improve start-up performance
      • reduce memory consumption
    • Package Explorer / Java Browsing
      • hierarchical package browsing
      • support logical packages
    • Type Hierarchy
      • support for sorting/grouping methods by defining interface/class
    • Java Editor
      • improved syntax highlighting/error feedback
      • improved navigation and typing support
      • user defined and controlled hover behavior
      • outline of compilation unit in a content assist like lightweight window
      • improved overview ruler (hover support and more markers)
      • code assist: eating mode and improved method completion
    • Build Path
      • improved container support
      • flexible project layout (multiple output folder, excluded source folders, etc.)
    • Refactoring
      • lightweight wizard-dialog
      • improved extract method and pull-up refactoring
      • new refactorings (extract interface, change signature, inline method, etc.)
    • Code manipulation
      • support for code templates
      • member sorting support
      • generate delegate methods
      • more quick fixes

Getting the sources
  • Define a CVS connection to server pserver:anonymous@dev.eclipse.org:/home/eclipse,
  • Check out the R2_1_maintenance branch of the project org.eclipse.jdt.ui. This branch reflects the current R2.1 maintenance effort.
  • Make sure to import the prerequisite plug-ins (usually from the latest SDK R2.1.x build). Prerequisite plug-ins are already referenced on the project build path ( .classpath file).
  • The sources of this project are either 1.3 or 1.4 compatible, however for 1.3 backward compatibility (Eclipse is primarily aimed for 1.3 runtimes), the compiler compliance setting should be set to 1.3.
