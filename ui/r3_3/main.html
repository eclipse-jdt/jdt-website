Overall 3.3 Plan
The 3.3 plan below is a proto plan and input from the community is very welcome
General
  • Themed bug fixing: Spend 3-4 days on each feature area to bring that feature up-to date: Browsing, Javadoc exporter, JAR exporter, Encapsulate Field, Extract Method, Drag and Drop...
Refactoring
  • Remove the requirement to save resources before executing a refactoring (149462)
  • Warn when refactoring derived files (147516)
  • Lightweight dialogs for refactoring (92322)
  • Finish implementation of refactoring-aware update during CVS synchronization
  • Fill gaps in refactoring scripting. Scripts for Move and rename package.
  • Improve existing refactorings
    • [>3.3] Bring the reorg refactorings in shape again: Drag and drop, Cut copy, paste
    • Update 'Extract local' and 'Extract constant' to use the AST rewrite infrastructure
  • New refactorings
    • [>3.3] Replace invocations
    • Introduce parameter object
Quick Fix, Quick Assist & Clean Up
  • Add formatting , organize import and sort members to 'Clean up'
  • Offer clean up profiles
  • [>3.3] More 'clean ups' like 'add/remove boxing', 'conversion between array variable declaration styles', 'remove/add all fully qualified type names'
  • [>3.3] Deprecation Quick Fix
  • Add local refactorings as quick assist with linked mode:
    • Extract local
    • Extract constant
    • Inline local
    • Convert local to field
    • Convert anonymous to member type
Code Manipulation and Generation
  • Improve Organize Imports to avoid some user prompting
  • Apply coding conventions (like formatting, organize import, nls, ...) on save , compile or commit.
  • [>3.3] New code style option to auto-generate parameters final
Build Path & New Wizards
  • Offer selection of a execution environment in the build path wizards
Java views (Package explorer, Type Hierarchy, Open Type)
  • Package explorer: Support better grouping mechanisms for referenced libraries (bug 83389)
  • Continue to improve the JDT part of the common navigator.
  • Multi-select in Open Type dialog
  • Improve type indicator performance
  • Use the new SWT capabilities in our views: different colors / styles in trees and tables
  • Simplify the managment of Java working sets: Improve 'Edit Java Working Set' dialog so it better deals with big workspaces (bug 154186)
Java Search
  • Support more Java search options
  • Investigate in advanced search with access to the AST. For example allow searching for all method calls to IRunnableContext#run where the first argument is boolean 'false'.
JUnit
  • Improve memory footprint of JUnit history
  • JUnit container to configure Javadoc location
  • [>3.3] Reimplement JUnit container to avoid plugin layout knowledge
  • Improve test run listener support (147261)
  • Improve test finder to find JUnit 4 tests in hierarchy
  • API for JUnitLaunchShortcut, ConfigurationTab and LaunchDelegate
API
  • [>3.3] API for the rendering of IBindings, similar to what JavaElementLabels does
  • Add API to invoke refactorings headless (without any UI dependency) (implemented for rename and move)
Miscellaneous JDT UI
  • [>3.3] Use trim to show number of error/warnings (prototype in jdt.ui)
  • [>3.3] Strings with basic LTR reading order not rendered correctly in RTL locales
  • [>3.3] Links in rich hovers: Links to help, links to preference pages
  • [>3.3] Prototype a change history view that shows all changes a refactoring performed
LTK
  • [>3.3] Move action handler ids down to LTK
    • Refactorings
    • Navigate Actions: Type Hierarchy, Call Hierarchy,
  • [>3.3] Investigate in common call hierarchy
  • [>3.3] Code audit view
Platform Search
  • [>3.3] Non modal replace dialog
  • [>3.3] Show line matches
  • [>3.3] Open search dialog from current search

Getting the sources
  • Define a CVS connection to server pserver:anonymous@dev.eclipse.org:/cvsroot/eclipse,
  • Check out HEAD branch of the project org.eclipse.jdt.ui.
  • Make sure to import the prerequisite plug-ins (usually from the latest SDK R3.2.x build). Prerequisite plug-ins are already referenced on the project build path ( .classpath file).
  • The sources of this project require JDK 1.4 or greater
