Overall 3.3 Plan
The 3.3 plan below is a proto plan and input from the community is very welcome
General
  • Themed bug fixing: Spend 3-4 days on each feature area to bring that feature up-to date: Browsing, Javadoc exporter, JAR exporter, Encapsulate Field, Extract Method, Drag and Drop...
Refactoring
  • Remove the requirement to save resources before executing a refactoring (149462)
  • Warn when refactoring derived files (147516)
  • Lightweight dialogs for refactoring (92322)
  • Finish implementation of refactoring-aware update during CVS synchronization
  • Fill gaps in refactoring scripting. Scripts for Move and rename package.
  • API
    • Add API to invoke all refactorings headless (without any UI dependency)
  • Improve existing refactorings
    • Bring the reorg refactorings in shape again: Drag and drop, Cut copy, paste
    • Update 'Extract local' and 'Extract constant' to use the AST rewrite infrastructure
  • New refactorings
    • Replace invocations
    • Introduce parameter object
Quick Fix, Quick Assist & Clean Up
  • Add formatting, organize import and sort members to 'Clean up'
  • Offer clean up profiles
  • More 'clean ups' like 'add/remove boxing', 'conversion between array variable declaration styles', 'remove/add all fully qualified type names'
  • Deprecation Quick Fix
Code Manipulation and Generation
  • Improve Organize Imports to avoid some user prompting
  • Apply coding conventions (like formatting, organize import, nls, ...) on save, compile or commit.
  • New code style option to auto-generate parameters final
Build Path & New Wizards
  • Offer selection of a execution environment in the build path wizards
Java views (Package explorer, Type Hierarchy, Open Type)
  • Package explorer: Support better grouping mechanisms for referenced libraries
  • Continue to improve the JDT part of the common navigator.
  • Multi-select in Open Type dialog
  • Improve type indicator performance
  • Use the new SWT capabilities in our views: different colors / styles in trees and tables
Java Search
  • Support more Java search options
  • Investigate in advanced search with access to the AST. For example allow searching for all method calls to IRunnableContext#run where the first argument is boolean 'false'.
JUnit
  • Improve memory footprint of JUnit history
  • JUnit container to configure Javadoc location
  • Reimplement JUnit container to avoid plugin layout knowledge
Miscellaneous JDT UI
  • Use trim to show number of error/warnings
  • Strings with basic LTR reading order not rendered correctly in RTL locales
  • Links in rich hovers: Links to help, links to preference pages
  • Prototype a change history view that shows all changes a refactoring performed
LTK
  • Move action handler ids down to LTK
    • Refactorings
    • Navigate Actions: Type Hierarchy, Call Hierarchy,
  • Investigate in common call hierarchy
  • Code audit view
Platform Search
  • Non modal replace dialog
  • Show line matches
  • Open search dialog from current search

Getting the sources
  • Define a CVS connection to server pserver:anonymous@dev.eclipse.org:/home/eclipse,
  • Check out HEAD branch of the project org.eclipse.jdt.ui.
  • Make sure to import the prerequisite plug-ins (usually from the latest SDK R3.1.x build). Prerequisite plug-ins are already referenced on the project build path ( .classpath file).
  • The sources of this project require JDK 1.4 or greater
