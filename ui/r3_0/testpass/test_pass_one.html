Refactoring

  • Test refactoring undo/redo
    • perform undo/redo with unsaved files not affected by the undo
    • perform undo/redo with unsaved files effected by the undo.
    • perform build after refactoring and ensure that undo is still possible.
    • test executing multiple refactorings and then the corresponding undo/redo sequence
    • change Java element (e.g save file): ensure that undo stack gets flushed.
  • Test malicious refactoring participants
    • install plug-in ....
    • test one rename, move and delete refactoring using the following steps
      • press OK on the input page
      • you get an error dialog informing that a participant caused an exception. Make sure that the refactoring didn't change the workspace
      • execute refactoring again
      • you get an error dialog saying that the change execution failed. Press Undo on the error dialog. Make sure that workspace is reverted back to its original state
      • execute refactoring again. This time the refactoring should pass since failing participants get disabled for the rest of the session.
