Pseudo Bug Keywords

To simplify bug management we started to tag JDT UI bugs with additional pseudo keywords (not normal Bugzilla keywords). The tags are appended to the bug's summary field. Since we use these tags for internal bug management reporters of a bug should not add any pseudo keywords when filing the bug. The owner of the component bucket is responsible to add the keywords.

Keywords are used to group bugs without assigning them to a developer. So with the introduction of the keywords it is easy to search for all bugs belonging to a specific sub component. For example to get an overview of all open refactoring issues search for new, assigned or resolved-later bugs containing the word [refactoring] in the summary field.

Keyword
Description
[api] API bugs (wrong or missing API)
[browsing] Bugs related to the Java Browsing perspective
[build path] Bugs related to the Java Build Path (Properties->Java Build Path)
[ccp] Copy, Cut, Paste bugs
[code manipulation] Bugs related to code manipulation / generation functionality like new wizards, source menu actions, ...
[compare] Bugs related to Java compare functionality. Not used to tag general compare bugs
[dialogs] Bugs related to general Java dialogs like open type dialog, browse packages dialog, ...
[dnd] Drag and Drop bugs. This keyword is mostly used together with other keywords like [refactoring] [type hierarchy], ...
[export/import] Bugs related to JDT's export/import functionality
[filters] Bugs related to JDT's filter support
[general issue] Bugs which manifest themselves in JDT but can't be solve by JDT only. The are related to problems in the platform or JDT/Core
[icons] Strange looking or missing icons
[jar creation] Bugs related to the Jar export functionality
[javadoc] Javadoc bugs either in the Javadoc export wizard or in Javadoc code assist
[JUnit] Bugs related to the org.eclipse.jdt.junit plug-in
[misc]  
[package explorer] Bugs related to JDT's package explorer
[quick fix] Bugs related to the Quick Fix functionality
[refactoring] Refactoring bugs including Undo/Redo management in refactoring.
[type hierarchy] Bugs related to the Type Hierarchy view
[search] Java search bugs. Not used to tag general search bugs.
[render] Bugs related to rendering of Java elements
[wording] Spelling and bogus wording
