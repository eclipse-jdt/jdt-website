JDT proposed plan items for release 3.5


Java 7 support

Module (JSR 177/194), more annotations, annotations everywhere (JSR 305/308)

Pre-processing

More flexible classpath layout

When adopting Eclipse, some pre-existing project layouts are hard to model in Eclipse:
  • multiple source attachments per library
  • relax classpath container limitations, which currently do not allow nested containers, source folders etc...
  • chained JARs
  • nested JARs
  • remove limitation on read-only external classfolder
  • pluggable archive reader (e.g. RAR support)
  • macro-component on buildpath: a la Jazz

Finer granularity of classpath and compiler settings

Allow more configurability of the Java project, such as allowing custom buildpath or settings on a per source folder basis

Better experience out of the box

  • Couple compiler compliance and JRE setting
  • Better integration with project explorer (http://www.codecommit.com/blog/eclipse/the-problem-of-perspective-multiplicity)
  • Merge local history and CVS history, offer comparisons (sync) even if offline
  • Provide User Explanation and User Action for Messages
  • Provide (extensible) mechanism to convert a binary project back into master source form

Editor improvements

  • Indent lines in control flow statements (84878). Richer compare editor experience.

Performance

  • Faster plug-in launch configs

Search improvement

  • Integrate websearch into Java search (http://eclipse-projects.blogspot.com/2008/05/google-is-cool.html)

Refactoring

  • Refactor signaling ramifications in binary dependents (producing refactoring scripts for these?)

Code formatting

  • Preserve line breaks
